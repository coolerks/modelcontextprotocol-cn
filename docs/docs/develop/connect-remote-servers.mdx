远程 MCP server 将 AI 应用程序的能力扩展到本地环境之外，提供对互联网托管的工具、服务和数据源的访问。通过连接到远程 MCP server，你将 AI 助手从有用的工具转变为信息丰富的队友，能够处理复杂的多步骤项目，并实时访问外部资源。

许多 clients 现在支持远程 MCP server，实现了广泛的集成可能性。本指南以 [Claude](https://claude.ai/) 为例演示如何连接到远程 MCP server，Claude 是众多支持 MCP 的 clients 之一。虽然我们专注于通过 Custom Connectors 实现 Claude 的集成，但这些概念广泛适用于其他 MCP 兼容 clients。

## 理解远程 MCP Server

远程 MCP server 的功能类似于本地 MCP server，但托管在互联网上而非你的本地机器上。它们暴露 tools、prompts 和 resources，Claude 可以用它们来代表你执行任务。这些 servers 可以与各种服务集成，如项目管理工具、文档系统、代码仓库和任何其他支持 API 的服务。

远程 MCP server 的关键优势在于其可访问性。与需要在每台设备上安装和配置的本地 servers 不同，远程 servers 可以通过任何具有互联网连接的 MCP client 访问。这使得它们非常适合基于 Web 的 AI 应用程序、强调易用性的集成以及需要服务器端处理或认证的服务。

## 什么是 Custom Connectors？

Custom Connectors 充当 Claude 和远程 MCP server 之间的桥梁。它们允许你将 Claude 直接连接到对你工作流最重要的工具和数据源，使 Claude 能够在你最喜欢的软件中操作，并从你的外部工具的完整上下文中获取洞察。

使用 Custom Connectors，你可以：

- [将 Claude 连接到第三方开发者提供的现有远程 MCP server](https://support.anthropic.com/en/articles/11175166-getting-started-with-custom-connectors-using-remote-mcp)
- [构建你自己的远程 MCP server 以连接任何工具](https://support.anthropic.com/en/articles/11503834-building-custom-connectors-via-remote-mcp-servers)

## 连接到远程 MCP Server

将 Claude 连接到远程 MCP server 的过程涉及通过 [Claude 界面](https://claude.ai/)添加 Custom Connector。这将在 Claude 和你选择的远程 server 之间建立安全连接。

<Steps>
<Step title="导航到 Connector 设置">
在浏览器中打开 Claude 并导航到设置页面。你可以通过点击个人资料图标并从下拉菜单中选择"Settings"来访问。进入设置后，找到并点击侧边栏中的"Connectors"部分。

这将显示你当前配置的 connectors，并提供添加新 connectors 的选项。

</Step>

<Step title="添加 Custom Connector">
在 Connectors 部分，滚动到底部，你会找到"Add custom connector"按钮。点击此按钮开始连接过程。

<Frame>
  <img
    src="/images/quickstart-remote/1-add-connector.png"
    alt="Claude 设置中的添加自定义 connector 按钮"
  />
</Frame>

将出现一个对话框，提示你输入远程 MCP server URL。此 URL 应由 server 开发者或管理员提供。输入完整的 URL，确保包含正确的协议（https://）和任何必要的路径组件。

<Frame>
  <img
    src="/images/quickstart-remote/2-connect.png"
    alt="输入远程 MCP server URL 的对话框"
  />
</Frame>

输入 URL 后，点击"Add"继续连接。

</Step>

<Step title="完成认证">
大多数远程 MCP server 需要认证以确保对其资源的安全访问。认证过程因 server 实现而异，但通常涉及 OAuth、API keys 或用户名/密码组合。

<Frame>
  <img
    src="/images/quickstart-remote/3-auth.png"
    alt="远程 MCP server 的认证界面"
  />
</Frame>

按照 server 提供的认证提示进行操作。这可能会将你重定向到第三方认证提供商，或在 Claude 内显示表单。认证完成后，Claude 将建立与远程 server 的安全连接。

</Step>

<Step title="访问 Resources 和 Prompts">
成功连接后，远程 server 的 resources 和 prompts 将在你的 Claude 对话中可用。你可以通过点击消息输入区域中的回形针图标来访问它们，这将打开附件菜单。

<Frame>
  <img
    src="/images/quickstart-remote/4-select-resources-menu.png"
    alt="显示可用 resources 的附件菜单"
  />
</Frame>

菜单显示来自已连接 servers 的所有可用 resources 和 prompts。选择你想要包含在对话中的项目。这些 resources 为 Claude 提供来自你外部工具的上下文和信息。

<Frame>
  <img src="/images/quickstart-remote/5-select-prompts-resources.png" alt="从菜单中选择特定的 resources 和 prompts" />
</Frame>
</Step>

<Step title="配置 Tool 权限">
远程 MCP server 通常暴露多个具有不同能力的 tools。你可以通过在 connector 设置中配置权限来控制允许 Claude 使用哪些 tools。这确保 Claude 只执行你明确授权的操作。

<Frame>
  <img
    src="/images/quickstart-remote/6-configure-tools.png"
    alt="Tool 权限配置界面"
  />
</Frame>

导航回 Connectors 设置并点击你已连接的 server。在这里，你可以根据需求启用或禁用特定的 tools、设置使用限制以及配置其他安全参数。

</Step>
</Steps>

## 使用远程 MCP Server 的最佳实践

在使用远程 MCP server 时，请考虑以下建议以确保安全高效的体验：

**安全考虑**：在连接前始终验证远程 MCP server 的真实性。仅连接到来自可信来源的 servers，并审查认证期间请求的权限。谨慎授予对敏感数据或系统的访问权限。

**管理多个 connectors**：你可以同时连接到多个远程 MCP servers。按用途或项目组织你的 connectors 以保持清晰。定期审查并删除不再使用的 connectors，以保持工作区的组织和安全。
