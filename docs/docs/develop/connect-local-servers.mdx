Model Context Protocol (MCP) server 通过提供对本地资源和工具的安全、受控访问来扩展 AI 应用程序的能力。许多 clients 支持 MCP，实现了跨不同平台和应用程序的多样化集成可能性。

本指南以 Claude Desktop 为例演示如何连接到本地 MCP server，Claude Desktop 是众多支持 MCP 的 clients 之一。虽然我们专注于 Claude Desktop 的实现，但这些概念广泛适用于其他 MCP 兼容 clients。通过本教程，Claude 将能够与你计算机上的文件进行交互、创建新文档、组织文件夹以及搜索文件系统——所有操作都需要你明确授权。

## 前提条件

在开始本教程之前，请确保你的系统已安装以下内容：

### Claude Desktop

为你的操作系统下载并安装 [Claude Desktop](https://claude.ai/download)。Claude Desktop 适用于 macOS 和 Windows。

如果你已经安装了 Claude Desktop，请通过点击 Claude 菜单并选择"检查更新..."来验证你是否运行的是最新版本。

### Node.js

文件系统 Server 和许多其他 MCP server 需要 Node.js 才能运行。打开终端或命令提示符并运行以下命令验证你的 Node.js 安装：

```bash
node --version
```

如果未安装 Node.js，请从 [nodejs.org](https://nodejs.org/) 下载。我们建议使用 LTS（长期支持）版本以获得稳定性。

## 理解 MCP Server

MCP server 是在你的计算机上运行的程序，通过标准化协议向 Claude Desktop 提供特定能力。每个 server 暴露 Claude 可以用来执行操作的 tools，并需要你的批准。我们将安装的文件系统 Server 提供以下 tools：

- 读取文件内容和目录结构
- 创建新文件和目录
- 移动和重命名文件
- 按名称或内容搜索文件

所有操作在执行前都需要你的明确批准，确保你对 Claude 可以访问和修改的内容保持完全控制。

## 安装文件系统 Server

该过程涉及配置 Claude Desktop，使其在每次启动应用程序时自动启动文件系统 Server。此配置通过一个 JSON 文件完成，该文件告诉 Claude Desktop 要运行哪些 servers 以及如何连接到它们。

<Steps>
<Step title="打开 Claude Desktop 设置">
首先访问 Claude Desktop 设置。点击系统菜单栏中的 Claude 菜单（不是 Claude 窗口内的设置），然后选择"Settings..."

在 macOS 上，这出现在顶部菜单栏中：

<Frame style={{ textAlign: "center" }}>
  <img
    src="/images/quickstart-menu.png"
    width="400"
    alt="显示设置选项的 Claude Desktop 菜单"
  />
</Frame>

这将打开 Claude Desktop 配置窗口，该窗口与你的 Claude 账户设置是分开的。

</Step>

<Step title="访问开发者设置">
在设置窗口中，导航到左侧边栏的"Developer"选项卡。此部分包含配置 MCP server 和其他开发者功能的选项。

点击"Edit Config"按钮打开配置文件：

<Frame>
  <img
    src="/images/quickstart-developer.png"
    alt="显示 Edit Config 按钮的开发者设置"
  />
</Frame>

如果配置文件不存在，此操作将创建一个新文件，或者打开你现有的配置。该文件位于：

- **macOS**：`~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**：`%APPDATA%\Claude\claude_desktop_config.json`

</Step>

<Step title="配置文件系统 Server">
将配置文件的内容替换为以下 JSON 结构。此配置告诉 Claude Desktop 启动文件系统 Server，并授予其对特定目录的访问权限：

<CodeGroup>

```json macOS
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/username/Desktop",
        "/Users/username/Downloads"
      ]
    }
  }
}
```

```json Windows
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "C:\\Users\\username\\Desktop",
        "C:\\Users\\username\\Downloads"
      ]
    }
  }
}
```

</CodeGroup>

将 `username` 替换为你实际的计算机用户名。`args` 数组中列出的路径指定了文件系统 Server 可以访问的目录。你可以根据需要修改这些路径或添加其他目录。

<Tip>
**理解配置**

- `"filesystem"`：在 Claude Desktop 中显示的 server 友好名称
- `"command": "npx"`：使用 Node.js 的 npx 工具运行 server
- `"-y"`：自动确认 server 包的安装
- `"@modelcontextprotocol/server-filesystem"`：文件系统 Server 的包名
- 其余参数：Server 被允许访问的目录

</Tip>

<Warning>
**安全考虑**

仅授予你愿意让 Claude 读取和修改的目录的访问权限。Server 使用你的用户账户权限运行，因此它可以执行你可以手动执行的任何文件操作。

</Warning>
</Step>

<Step title="重启 Claude Desktop">
保存配置文件后，完全退出 Claude Desktop 并重新启动。应用程序需要重新启动以加载新配置并启动 MCP server。

成功重启后，你将在对话输入框的右下角看到一个 MCP server 指示器 <img src="/images/claude-desktop-mcp-slider.svg" style={{display: 'inline', margin: 0, height: '1.3em'}} />：

<Frame>
  <img
    src="/images/quickstart-slider.png"
    alt="显示 MCP server 指示器的 Claude Desktop 界面"
  />
</Frame>

点击此指示器查看文件系统 Server 提供的可用 tools：

<Frame style={{ textAlign: "center" }}>
  <img
    src="/images/quickstart-tools.png"
    width="400"
    alt="Claude Desktop 中可用的文件系统 tools"
  />
</Frame>

如果 server 指示器没有出现，请参阅[故障排除](#troubleshooting)部分进行调试。

</Step>
</Steps>

## 使用文件系统 Server

连接文件系统 Server 后，Claude 现在可以与你的文件系统交互。尝试以下示例请求来探索其能力：

### 文件管理示例

- **"你能写一首诗并保存到我的桌面上吗？"** - Claude 将创作一首诗并在你的桌面上创建一个新的文本文件
- **"我的下载文件夹中有哪些工作相关的文件？"** - Claude 将扫描你的下载文件夹并识别工作相关的文档
- **"请将我桌面上的所有图片整理到一个名为'Images'的新文件夹中"** - Claude 将创建一个文件夹并将图片文件移入其中

### 批准机制

在执行任何文件系统操作之前，Claude 将请求你的批准。这确保你对所有操作保持控制：

<Frame style={{ textAlign: "center" }}>
  <img
    src="/images/quickstart-approve.png"
    width="500"
    alt="Claude 请求批准以执行文件操作"

在批准前仔细审查每个请求。如果你对建议的操作不满意，可以随时拒绝。

## 故障排除

如果在设置或使用文件系统 Server 时遇到问题，以下解决方案可解决常见问题：

<AccordionGroup>
<Accordion title="Server 未在 Claude 中显示 / 锤子图标缺失">

1. 完全重启 Claude Desktop
2. 检查你的 `claude_desktop_config.json` 文件语法
3. 确保 `claude_desktop_config.json` 中包含的文件路径是有效的，并且是绝对路径而非相对路径
4. 查看[日志](#getting-logs-from-claude-for-desktop)了解 server 未连接的原因
5. 在命令行中，尝试手动运行 server（像在 `claude_desktop_config.json` 中那样替换 `username`），查看是否有任何错误：

<CodeGroup>

```bash macOS/Linux
npx -y @modelcontextprotocol/server-filesystem /Users/username/Desktop /Users/username/Downloads
```

```powershell Windows
npx -y @modelcontextprotocol/server-filesystem C:\Users\username\Desktop C:\Users\username\Downloads
```

</CodeGroup>

</Accordion>
<Accordion title="从 Claude Desktop 获取日志">

与 MCP 相关的 Claude.app 日志写入以下日志文件：

- macOS：`~/Library/Logs/Claude`
- Windows：`%APPDATA%\Claude\logs`

- `mcp.log` 包含关于 MCP 连接和连接失败的一般日志。
- 名为 `mcp-server-SERVERNAME.log` 的文件包含来自指定 server 的错误（stderr）日志。

你可以运行以下命令列出最近的日志并跟踪任何新日志（在 Windows 上，它只会显示最近的日志）：

<CodeGroup>

```bash macOS/Linux
tail -n 20 -f ~/Library/Logs/Claude/mcp*.log
```

```powershell Windows
type "%APPDATA%\Claude\logs\mcp*.log"
```

</CodeGroup>

</Accordion>
<Accordion title="Tool 调用静默失败">

如果 Claude 尝试使用 tools 但失败：

1. 检查 Claude 的日志中的错误
2. 验证你的 server 构建和运行没有错误
3. 尝试重启 Claude Desktop

</Accordion>
<Accordion title="这些都不管用。我该怎么办？">

请参考我们的[调试指南](/docs/tools/debugging)获取更好的调试工具和更详细的指导。

</Accordion>
<Accordion title="Windows 上的 ENOENT 错误和路径中的 `${APPDATA}`">

如果你配置的 server 加载失败，并且在日志中看到路径中引用了 `${APPDATA}` 的错误，你可能需要在 `claude_desktop_config.json` 的 `env` 键中添加 `%APPDATA%` 的展开值：

```json
{
  "brave-search": {
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-brave-search"],
    "env": {
      "APPDATA": "C:\\Users\\user\\AppData\\Roaming\\",
      "BRAVE_API_KEY": "..."
    }
  }
}
```

完成此更改后，再次启动 Claude Desktop。

<Warning>

**npm 应全局安装**

如果你尚未全局安装 npm，`npx` 命令可能继续失败。如果 npm 已经全局安装，你会在系统上找到 `%APPDATA%\npm`。如果没有，你可以通过运行以下命令全局安装 npm：

```bash
npm install -g npm
```

</Warning>

</Accordion>
</AccordionGroup>
