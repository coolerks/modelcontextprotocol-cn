[Agent skills](https://agentskills.io/home) 是可移植的指令集，为 AI 编码助手提供特定任务的领域知识。对于 MCP 开发，它们将设计决策（部署模型、tool 模式、认证）编码，使你的 agent 能够询问你的用例并搭建适合的 server。

## 可用 skills

一组参考 MCP 开发 skills 可作为
[`mcp-server-dev` 插件](https://github.com/anthropics/claude-plugins-official/tree/main/plugins/mcp-server-dev)获取。
它提供了三个组合 skills：

| Skill              | 用途                                                                     |
| ------------------ | ------------------------------------------------------------------------ |
| `build-mcp-server` | 入口点。询问用例，选择部署模型和 tool 设计模式，路由到专门 skills。      |
| `build-mcp-app`    | 添加在聊天中内联渲染的交互式 UI 组件（表单、选择器、仪表板）。           |
| `build-mcpb`       | 将本地 stdio server 与其运行时打包，使用户无需 Node 或 Python 即可安装。 |

每个 skill 附带一个 `SKILL.md` 文件以及一个 `references/` 文件夹，内含支持材料（认证流程、tool 设计模式、组件模板、manifest 模式），agent 按需读取。这些文件遵循开放格式，适用于任何实现该标准的 agent。例如，要在 Claude Code 中安装它们：

```bash
/plugin marketplace add anthropics/claude-plugins-official
/plugin install mcp-server-dev
```

对于其他 agents，请查看你的 skills 或扩展目录，或将
[skill 目录](https://github.com/anthropics/claude-plugins-official/tree/main/plugins/mcp-server-dev/skills)
（`SKILL.md` 加上 `references/`）克隆到你的 agent 的 skills 位置。

## 开始构建

安装 skills 后，请你的 agent 帮助你构建 MCP server。入口 skill 在自然语言请求时触发，或者你可以使用 agent 的 skill 调用语法直接调用它。

该 skill 在编写任何代码之前会运行一个简短的发现阶段。可能会问到以下问题：

- **它连接什么** — 云 API、本地进程、文件系统、硬件
- **谁将使用它** — 仅你、你的团队，还是任何安装它的人
- **操作面大小** — 少量操作还是包装大型 API
- **用户交互需求** — 纯文本结果、通过 [elicitation](/specification/draft/client/elicitation) 的结构化输入，或丰富的 UI 组件
- **上游认证** — API keys、OAuth 2.0 或无

如果你的开场消息已经涵盖了这些内容，agent 将直接跳到建议阶段。

## 部署路径

根据发现结果，该 skill 会推荐以下四条路径之一并进行相应搭建：

**远程 [Streamable HTTP](/specification/draft/basic/transports/streamable-http)**
是包装云 API 的默认选项。零安装摩擦，一次部署服务于所有用户，OAuth 流程正常工作，因为 server 可以处理重定向和令牌存储。参考 skill 包括 Cloudflare Workers 和可移植 Express/FastMCP 设置的模板。

**[MCP 应用](/extensions/apps/overview)** 通过交互式小部件扩展 server，这些小部件在聊天中渲染，如可搜索的选择器、图表和实时仪表板。当 [elicitation 的](/specification/draft/client/elicitation) 平面表单约束不适用时，该 skill 移交给 `build-mcp-app`。

**[MCP Bundles (MCPB)](https://github.com/modelcontextprotocol/mcpb)** 将本地 server 与其运行时打包为单个 `.mcpb` 归档文件，使用户无需设置 Node 或 Python 即可安装。当 server 必须接触用户机器时使用此路径：读取本地文件、驱动桌面应用程序或与 localhost 服务通信。该 skill 移交给 `build-mcpb`。

**本地 [stdio](/specification/draft/basic/transports/stdio)** 仍然可用于原型设计，并附有明确的升级路径，在准备分发时迁移到 MCPB。
