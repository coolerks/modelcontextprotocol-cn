[MCP Inspector](https://github.com/modelcontextprotocol/inspector) 是一个用于测试和调试 MCP server 的交互式开发者工具。[调试指南](/docs/tools/debugging) 将 Inspector 作为整体调试工具包的一部分进行了介绍，而本文档则详细探讨了 Inspector 的功能和能力。

## 开始使用

### 安装与基本用法

Inspector 直接通过 `npx` 运行，无需安装：

```bash
npx @modelcontextprotocol/inspector <command>
```

```bash
npx @modelcontextprotocol/inspector <command> <arg1> <arg2>
```

#### 检查来自 npm 或 PyPI 的 servers

从 [npm](https://npmjs.com) 或 [PyPI](https://pypi.org) 启动 server 包的常见方式。

<Tabs>

  <Tab title="npm 包">

```bash
npx -y @modelcontextprotocol/inspector npx <package-name> <args>
# 例如
npx -y @modelcontextprotocol/inspector npx @modelcontextprotocol/server-filesystem /Users/username/Desktop
```

  </Tab>

  <Tab title="PyPI 包">

```bash
npx @modelcontextprotocol/inspector uvx <package-name> <args>
# 例如
npx @modelcontextprotocol/inspector uvx mcp-server-git --repository ~/code/mcp/servers.git
```

  </Tab>
</Tabs>

#### 检查本地开发的 servers

要检查本地开发或作为仓库下载的 servers，最常见的方式是：

<Tabs>
  <Tab title="TypeScript">

```bash
npx @modelcontextprotocol/inspector node path/to/server/index.js args...
```

  </Tab>
  <Tab title="Python">

```bash
npx @modelcontextprotocol/inspector \
  uv \
  --directory path/to/server \
  run \
  package-name \
  args...
```

  </Tab>
</Tabs>

请仔细阅读任何附带的 README 以获取最准确的说明。

## 功能概览

<Frame caption="MCP Inspector 界面">
  <img src="/images/mcp-inspector.png" />
</Frame>

Inspector 提供了多种与 MCP server 交互的功能：

### Server 连接面板

- 允许选择连接到 server 的[传输](/specification/latest/basic/transports)
- 对于本地 servers，支持自定义命令行参数和环境

### Resources 选项卡

- 列出所有可用 resources
- 显示 resource 元数据（MIME 类型、描述）
- 允许检查 resource 内容
- 支持订阅测试

### Prompts 选项卡

- 显示可用的 prompt 模板
- 显示 prompt 参数和描述
- 支持使用自定义参数测试 prompt
- 预览生成的消息

### Tools 选项卡

- 列出可用 tools
- 显示 tool schema 和描述
- 支持使用自定义输入测试 tool
- 显示 tool 执行结果

### 通知面板

- 呈现从 server 记录的所有日志
- 显示从 server 收到的通知

## 最佳实践

### 开发工作流

1. 开始开发
   - 使用你的 server 启动 Inspector
   - 验证基本连接
   - 检查能力协商

2. 迭代测试
   - 修改 server
   - 重新构建 server
   - 重新连接 Inspector
   - 测试受影响的功能
   - 监控消息

3. 测试边界情况
   - 无效输入
   - 缺少 prompt 参数
   - 并发操作
   - 验证错误处理和错误响应
