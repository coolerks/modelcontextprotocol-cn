本文档说明了如何在 Model Context Protocol (MCP) 项目中进行沟通和协作。

## 沟通渠道

| 渠道                                                                                           | 用途       | 何时使用                   |
| ---------------------------------------------------------------------------------------------- | ---------- | -------------------------- |
| [Discord](https://discord.gg/6CSzBmMkjX)                                                       | 实时讨论   | 快速提问、协调、WG/IG 讨论 |
| [实时会议](https://meet.modelcontextprotocol.io/)                                              | 同步沟通   | WG/IG 演示、进度报告       |
| [GitHub Discussions](https://github.com/modelcontextprotocol/modelcontextprotocol/discussions) | 结构化讨论 | 提案、路线图规划、长篇辩论 |
| [GitHub Issues](https://github.com/modelcontextprotocol/modelcontextprotocol/issues)           | 可操作任务 | Bug 报告、文档修复         |
| [漏洞报告](https://github.com/modelcontextprotocol/modelcontextprotocol/blob/main/SECURITY.md) | 安全问题   | 漏洞 - **切勿公开发布**    |

所有沟通均受我们的[行为准则](https://github.com/modelcontextprotocol/modelcontextprotocol/blob/main/CODE_OF_CONDUCT.md)约束。我们期望在所有渠道中进行尊重、专业和包容的互动。

## Discord

[MCP 贡献者 Discord](https://discord.gg/6CSzBmMkjX) 用于实时贡献者讨论和协作。该服务器专为 **MCP 贡献者**设计，不用于一般的 MCP 支持。

### 公开频道（默认）

**目的：** 开放的社区参与、协作开发和透明的项目协调。

**主要用例：**

- SDK 和工具开发（例如 `#typescript-sdk-dev`、`#inspector-dev`）
- [工作组和兴趣小组](/community/working-interest-groups)讨论（例如 `#auth-wg`、`#security-ig`）
- 社区入职和贡献指导
- 社区反馈和协作头脑风暴
- 公开办公时间和维护者可用性

**避免：**

- MCP 用户支持 - 阅读官方文档并使用 GitHub Discussions 提问
- 服务或产品营销 - 保持讨论的中立性；不鼓励提及品牌，除非作为与规范相关的示例

### 私密频道（例外情况）

**目的：** 保密协调和敏感事务。访问权限仅限于指定的维护者。

**私密使用的标准：**

- 安全事件（CVE、协议漏洞）
- 人事事务（维护者讨论、行为准则问题）
- 需要立即或集中响应且受众有限的协调
- 某些频道对维护者决策为只读

**透明度要求：**

- 所有影响社区的技术和治理决策必须记录在 GitHub Discussions 和/或 Issues 中，并标记 `notes`
- 私密频道是临时的"事件处理室"，不用于日常开发
- 与个人贡献者相关的某些事项在适当时可以保密

任何在 Discord 上导致潜在决策或提案的重要讨论都必须移至 GitHub Discussion 或 Issue，以形成持久、可搜索的记录。

## GitHub Discussions

用于关于项目方向的结构化、长篇讨论和辩论。

**何时使用：**

- 项目路线图规划和里程碑讨论
- 公告和发布沟通
- 社区投票和共识建立
- 带有背景和理由的功能请求
- 如果仓库未启用 Discussions，请改用 GitHub Issues

## GitHub Issues

用于 Bug 报告和可操作的开发任务。功能请求应提交到 [GitHub Discussions](https://github.com/modelcontextprotocol/modelcontextprotocol/discussions)。

**何时使用：**

- 带有可重现步骤的 Bug 报告
- 具有特定范围的文档改进
- CI/CD 问题和基础设施问题
- 发布任务和里程碑跟踪

**注意：** SEP 提案是作为拉取请求提交到 [`seps/` 目录](https://github.com/modelcontextprotocol/modelcontextprotocol/tree/main/seps)，而不是作为 GitHub Issues。请参阅 [SEP 指南](/community/sep-guidelines)。

## 安全问题

**请勿公开发布安全问题。**

1. 使用 [SECURITY.md](https://github.com/modelcontextprotocol/modelcontextprotocol/blob/main/SECURITY.md) 中的私密安全报告流程
2. 直接联系主导者或[核心维护者](/community/governance#current-core-maintainers)
3. 遵循负责任的披露指南

## 决策记录

所有 MCP 决策都记录在公开渠道中：

| 类型     | 位置                                                                                        |
| -------- | ------------------------------------------------------------------------------------------- |
| 技术决策 | [GitHub Issues](https://github.com/modelcontextprotocol/modelcontextprotocol/issues) 和 SEP |
| 规范变更 | [变更日志](https://modelcontextprotocol.io/specification/draft/changelog)                   |
| 流程变更 | [社区文档](https://modelcontextprotocol.io/community/governance)                            |
| 治理决策 | [GitHub Issues](https://github.com/modelcontextprotocol/modelcontextprotocol/issues) 和 SEP |

在记录决策时，我们尽可能保留更多上下文：

- 决策者
- 背景和动机
- 考虑的选项
- 选择方案的理由
- 实施步骤
