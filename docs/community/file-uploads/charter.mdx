## 小组类型

**工作组**

## 使命宣言

文件上传工作组旨在定义 MCP 工具和引导请求如何声明文件输入，以便主机可以呈现原生文件选择器并将用户选择的文件内容传递给服务器。
目前，需要从用户获取文件的服务器只能通过文字说明要求 base64 字符串或本地路径，
这产生了不一致的用户体验并将编码细节推给了最终用户。本工作组
将指定一个最小化的、模式级别的机制来声明文件输入及其传输的线路格式，
基于 [SEP-2356](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/2356)。

## 范围

### 范围内

- **规范工作**：定义工具输入模式和引导请求模式上的声明性文件输入描述符、
  文件内容的线路编码以及主机端处理要求的 SEP。
- **参考实现**：用于 `FileInputDescriptor`、数据 URI 编码的 SDK 类型和助手，
  以及演示选择器调用和值替换的示例主机流程。
- **跨领域问题**：与 MCP Apps WG 的协调（其中嵌入式 UI 表面呈现自己的文件选择器），
  以及与安全 WG 的协调（主机端验证要求）。
- **文档**：涵盖文件输入声明的规范部分，以及为当前使用临时 base64 指令的服务器提供的迁移指南。

### 范围外

- 服务器到客户端的文件传递，这已被资源和 `BlobResourceContents` 覆盖。
- 对传输层或会话模型的更改。

WG 可以评估流式传输、分块传输或预签名上传 URL 等方法作为其设计工作的一部分；
这些方法是在初始 SEP 还是后续 SEP 中落地是 WG 的决定，而非章程约束。

### 相关小组

- **MCP Apps WG** — 嵌入式应用 UI 可能呈现自己的文件选择器；描述符格式应在此上下文中可重用。
- **安全 WG** — 用户提供的文件内容的主机端验证要求（SEP 引用 [OWASP ASVS V5](https://owasp.org/www-project-application-security-verification-standard/) 作为一般上传卫生标准）。
- **工具注释 IG** — 文件输入描述符是一种输入参数元数据形式，应与更广泛的注释分类保持一致。
