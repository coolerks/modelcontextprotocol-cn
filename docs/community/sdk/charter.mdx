## 小组类型

**工作组**

## 使命宣言

SDK 工作组旨在保持官方 MCP SDK 与规范的一致性、合规性和同步性。它协调新协议版本在多种语言中的实现，管理 [SDK 分层系统](/community/sdk-tiers)，并在合理的情况下建立共享设计模式，以便开发者在不同 SDK 之间获得一致的体验，同时每个 SDK 仍保持其语言的惯用风格。

## 范围

### 范围内

- **SDK 分层**：运营 [SDK 分层系统](/community/sdk-tiers)，包括评审层级晋升请求、应用降级标准，以及维护已发布的层级分配。
- **官方 SDK 名单**：评估新增官方 SDK 或淘汰现有 SDK 的提案。
- **发布协调**：使第 1 层 SDK 发布计划与规范版本日期对齐，以便协议功能在其层级要求的时间线上落地到 SDK。
- **跨 SDK 设计指导**：推荐 SDK API 表面、版本控制、弃用、错误处理和扩展打包的通用模式，使 SDK 在多种语言中保持可识别的相似性，同时保持惯用风格。
- **一致性集成**：与一致性测试项目合作，确保每个官方 SDK 运行一致性套件并发布结果。
- **维护者协调**：提供一个论坛，供各语言 SDK 维护者分享实现经验，并将规范歧义反馈给核心维护者。

### 范围外

- **各 SDK 日常维护**：单个 SDK 的问题分类、PR 评审和发布仍由该 SDK 的维护者负责，如 [MAINTAINERS.md](https://github.com/modelcontextprotocol/modelcontextprotocol/blob/main/MAINTAINERS.md) 中所列。
- **规范编写**：协议变更通过 [SEP 流程](/community/sep-guidelines) 提出，由相关工作组或核心维护者拥有。SDK WG 实现已接受的 SEP；它不拥有规范章节。
- **一致性测试编写**：一致性测试套件本身由 [Conformance Testing](https://github.com/modelcontextprotocol/conformance) 项目拥有。
- **第三方和社区 SDK**：[modelcontextprotocol](https://github.com/modelcontextprotocol) 组织之外的 SDK 不受本小组管理。

### 相关小组

- **传输工作组**：传输实现是每个 SDK 的重要组成部分。SDK WG 在传输 SEP 落地时与传输工作组协调发布顺序。
- **一致性测试**：层级分配取决于一致性评分。SDK WG 使用一致性结果并反馈测试覆盖的差距。
- **所有产生规范的工作组**：SDK WG 是已接受 SEP 的下游消费者，并与源工作组协调参考实现的时间安排。

## 权限与决策权

| 决策类型                        | 权限级别                             |
| ------------------------------- | ------------------------------------ |
| 会议后勤与安排                  | WG 负责人（自主）                    |
| WG 内提案优先级                 | WG 负责人（自主）                    |
| SDK 层级晋升或降级              | WG 共识                              |
| 跨 SDK 设计指导                 | WG 共识（对各 SDK 维护者具有建议性） |
| 各 SDK 发布、版本控制、API 设计 | 该 SDK 的维护者（自主）              |
| 添加或淘汰官方 SDK              | WG 共识 → 核心维护者批准             |
| 分层标准的变更                  | WG 共识 → 核心维护者批准             |
| 范围扩展                        | 需要核心维护者批准                   |
| WG 成员批准                     | WG 成员赞助者                        |

## 成员

WG 成员是各官方 SDK 的维护者，如 [MAINTAINERS.md](https://github.com/modelcontextprotocol/modelcontextprotocol/blob/main/MAINTAINERS.md) 和 [modelcontextprotocol/access](https://github.com/modelcontextprotocol/access) 中相应角色所记录。任何官方 SDK 的维护者默认即为 WG 成员。

## 运营

| 会议     | 频率       | 时长    | 目的                            |
| -------- | ---------- | ------- | ------------------------------- |
| 工作会议 | 每两周一次 | 45 分钟 | 发布协调、层级评审、跨 SDK 设计 |

沟通在 `#general-sdk-dev` Discord 频道和 [GitHub Discussions](https://github.com/modelcontextprotocol/modelcontextprotocol/discussions) 的 SDK 工作组类别中进行。

### 成功标准

- 所有官方 SDK 都有已发布的层级，并在其默认分支上通过一致性测试。
- 第 1 层 SDK 在其层级要求的时间线内，为每个已发布的规范版本提供支持。
- 层级晋升和降级决定记录在 GitHub Discussions 中，并附有理由。
