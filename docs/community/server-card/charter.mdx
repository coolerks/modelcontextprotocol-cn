## 小组类型

**工作组**

## 使命宣言

Server Card 工作组旨在定义通过通用发现机制促进 MCP 服务器发现和使用的机制，并就如何将其融入更广泛的 AI 标准化工作中关于发现的努力提供指导。具体来说，该工作组将定义 MCP Server Card 的构成、Server Card 必须遵循的标准化文档格式，以及客户端如何发现给定服务器的 Server Card。

## 范围

### 范围内

- **规范工作**：定义 MCP Server Card 构成和 Server Card 文档具体格式的 SEP 或扩展。
- **发现机制**：定义 MCP Server Card 文档如何被发现（well-known URL、基于资源的发现等）的规范。
- **跨生态系统协调**：向 [AI Card](https://github.com/Agent-Card/ai-card) 项目提供关于如何与 MCP Server Cards 交互的建议。
- **文档**：涵盖 Server Card 编写和使用的规范章节和指南。

### 范围外

- 对 MCP 初始化握手或传输层的更改。
- 通用 MCP 服务器注册中心或目录（由注册中心工作组负责）。
- 在定义 MCP 范围内的国际化方法之前，Server Card 内容的国际化。

### 相关小组

- **注册中心工作组** — Server Card 格式应尽可能保持为 `server.json` 的子集；需要协调以避免分歧。
- **AI Card 项目**（外部）—— 关于 AI 目录是否以及如何直接链接到 MCP Server Card 的持续讨论。
