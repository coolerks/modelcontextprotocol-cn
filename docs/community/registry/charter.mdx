## 小组类型

**工作组**

## 使命宣言

注册中心工作组旨在构建和维护官方 MCP 注册中心——一个公开可用的 MCP 服务器的开放目录和 API——以便客户端、子注册中心和最终用户可以自信地发现、评估和安装服务器。该工作组拥有注册中心服务、`server.json` 模式、注册中心 API 规范以及进一步分发服务器元数据的子注册中心生态系统。

## 范围

### 范围内

- **注册中心服务**：托管在 `registry.modelcontextprotocol.io` 的注册中心的运营、可靠性和演进，包括正常运行时间、监控和事件响应。
- **注册中心 API 规范**：定义任何注册中心（官方或私有）如何暴露服务器元数据的 OpenAPI 规范。
- **`server.json` 模式**：描述 MCP 服务器身份、包、运行时配置和能力的标准化格式——与 Server Card WG 协调，使 Server Card 保持一致子集。
- **客户端 SDK**：生成的或手动维护的客户端库，使客户端和子注册中心能够轻松与注册中心 API 集成。
- **发布与信任**：身份验证流程（GitHub OAuth、GitHub OIDC、DNS/HTTP 验证）、命名空间所有权、审核工具和社区驱动的标记。
- **采纳与推广**：文档、入门指南和推广活动，以提高目录覆盖率。
- **问题分类与自动化**：注册中心仓库的标签系统、分类和贡献者工作流程。

### 范围外

- 任何与运行时相关的 MCP 协议规范方面（由核心维护者和其他工作组负责）。
- Server Card 格式和发现机制（由 Server Card 工作组负责；本工作组在 `server.json` 对齐上进行协调）。
- 代表 MCP 客户端或最终用户在 MCP 服务器实现之间进行排名/选择。
- 托管、分发或执行 MCP 服务器代码或二进制文件——注册中心是一个元数据目录，而非包注册中心。
- 交付企业级或可复用的注册中心实现的任何承诺。代码库仅支持此实例，不适用于外部部署。

### 相关小组

- **Server Card WG** — `server.json` 和 Server Card 必须保持一致；注册中心将为已发布的条目暴露 Server Cards 和本地包相关元数据。需要紧密协调以避免模式分歧。

### 成功标准

- 注册中心正常运行时间 ≥ 99.9%，具有自动化监控和告警。
- `server.json` 模式和 Server Card 格式对齐，无意外的分歧。
- 大多数流行、公开可用的 MCP 服务器已发布到注册中心。
- 至少一个客户端 SDK（生成的或维护的）可供注册中心消费者使用。
- Registry API v1 规范最终确定并保持稳定。
- 活跃的子注册中心生态系统消费官方注册中心 API。
