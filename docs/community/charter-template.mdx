---

每个 MCP 工作组和兴趣小组必须维护一份遵循此结构的章程文档。章程存储在
[modelcontextprotocol 仓库](https://github.com/modelcontextprotocol/modelcontextprotocol) 的 `docs/community/<group-name>/charter.mdx` 中，
并添加到 `docs/docs.json`。

章程捕获特定于你的小组的信息。治理规则——领导层要求、决策流程、会议要求、升级路径——
在[工作组和兴趣小组](/community/working-interest-groups)文档中定义并自动适用。不要在此处重复它们。

标记为 **(WG only)** 的章节是工作组的必需内容，兴趣小组可选。

<Note>
  将下面的 markdown 复制到 `docs/community/<group-name>/charter.mdx` 中并替换占位文本。
</Note>

---

## 示例使命宣言

**工作组：**

> 传输工作组旨在发展 MCP 的传输机制，以支持多样化的部署场景——从本地子进程通信到水平扩展的云部署——同时保持协议的一致性和向后兼容性。

**兴趣小组：**

> 企业 IG 探索在企业环境中部署 MCP 的挑战，收集用例和需求，为未来的规范工作提供信息。
