## 使命宣言

Inspector V2 工作组正在构建 Inspector V2，这是一个全新的基于 Web 的 MCP 检查器，从头开始重新设计以追求可维护性和可靠性。该小组提供了一个共享的 Inspector Core 架构，最大化 Web、CLI 和 TUI 实现之间的代码复用，以及全面的测试设施。这项工作需要跨维护者协作，因为它涉及 UI、协议工具和测试基础设施，而这些目前不属于任何单个维护者的职责范围。

## 范围

### 范围内

- `modelcontextprotocol/inspector` 仓库，包括：
- **Inspector Core** — 一种新的共享代码架构，为所有 Inspector 前端提供通用 MCP 和协议接口。
- **Web Inspector UI** — 基于 Mantine 和 TypeScript 构建的浏览器检查器。
- **CLI Inspector** — 共享 Inspector Core 的命令行界面。
- **TUI Inspector** — 共享 Inspector Core 的终端 UI。
- **测试设施** — 跨所有 Inspector 表面的共享测试框架、测试夹具和集成测试。
- 将 `main` 上的现有 Inspector 弃用并迁移到 `v1.x` 维护分支。
- 一旦可用，在 Inspector Core 中采用 MCP TypeScript SDK V2。

### 范围外

- 核心 MCP 规范。
- MCP SDK 维护（TypeScript、Python 或任何其他语言 SDK）。
- MCP 服务器实现。

### 相关小组

- **SDK WG** — Inspector Core 使用 TypeScript SDK；需要协调 SDK V2 的采用。
- **MCP Apps WG** — 围绕客户端/应用人体工程学和检查工作流的共享表面区域。
- **Auth WG** — Inspector 在连接到受保护服务器时执行的身份验证流程。
- **Registry WG** — Inspector 向用户展示的发现和元数据表面。

5. **第二季度末** — `main` 上现有的 Inspector 移至 `v1.x` 分支并正式弃用。
6. **第二季度末** — 新的 Inspector 系列（Web、CLI、TUI）发布并普遍可用。
7. **第三季度末** — Inspector Core 在 MCP TypeScript SDK V2 上运行（由 TypeScript SDK WG 的交付计划决定）。
