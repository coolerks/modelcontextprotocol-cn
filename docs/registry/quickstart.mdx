<Note>
  MCP 注册表当前处于预览阶段。在正式发布之前可能发生破坏性变更或数据重置。如果您遇到任何问题，请在 [GitHub](https://github.com/modelcontextprotocol/registry/issues) 上报告。
</Note>

本教程将向您展示如何使用官方 `mcp-publisher` CLI 工具将用 TypeScript 编写的 MCP 服务器发布到 MCP 注册表。

## 前提条件

- **Node.js**——本教程假设 MCP 服务器是用 TypeScript 编写的。
- **npm 账户**——MCP 注册表仅托管元数据，而非制品。在发布到 MCP 注册表之前，我们需要将 MCP 服务器的包发布到 npm，因此您需要一个 [npm](https://www.npmjs.com) 账户。
- **GitHub 账户**——MCP 注册表支持[多种身份验证方法](./authentication)。为简单起见，本教程将使用基于 GitHub 的身份验证，因此您需要一个 [GitHub](https://github.com/) 账户。

如果您没有用 TypeScript 编写的 MCP 服务器，可以从 [`modelcontextprotocol/quickstart-resources` 仓库](https://github.com/modelcontextprotocol/quickstart-resources) 复制 `weather-server-typescript` 服务器来跟随本教程：

```bash
git clone --depth 1 git@github.com:modelcontextprotocol/quickstart-resources.git
cp -r quickstart-resources/weather-server-typescript .
rm -rf quickstart-resources
cd weather-server-typescript
```

然后编辑 `package.json` 以反映您的信息：

```diff package.json
 {
-  "name": "mcp-quickstart-ts",
-  "version": "1.0.0",
+  "name": "@my-username/mcp-weather-server",
+  "version": "1.0.1",
   "main": "index.js",
```

```diff package.json
   "license": "ISC",
-  "description": "",
+  "repository": {
+    "type": "git",
+    "url": "https://github.com/my-username/mcp-weather-server.git"
+  },
+  "description": "An MCP server for weather information.",
   "devDependencies": {
```

## 步骤 1：向包添加验证信息

MCP 注册表验证服务器的底层包是否与其元数据匹配。对于 npm 包，这需要在 `package.json` 中添加 `mcpName` 属性：

```diff package.json
 {
   "name": "@my-username/mcp-weather-server",
   "version": "1.0.1",
+  "mcpName": "io.github.my-username/weather",
   "main": "index.js",
```

`mcpName` 的值将是您的服务器在 MCP 注册表中的名称。

由于我们将使用基于 GitHub 的身份验证，`mcpName` **必须**以 `io.github.my-username/` 开头。

## 步骤 2：发布包

MCP 注册表仅托管元数据，而非制品，因此我们必须在将服务器发布到 MCP 注册表之前将包发布到 npm。

确保构建了分发文件：

```bash
# 导航到项目目录
cd weather-server-typescript

# 安装依赖
npm install

# 构建分发文件
npm run build
```

然后按照 npm 的[发布指南](https://docs.npmjs.com/creating-and-publishing-scoped-public-packages)操作。具体来说，您可能需要运行以下命令：

```bash
# 如有必要，向 npm 进行身份验证
npm adduser

# 发布包
npm publish --access public
```

您可以通过访问其 npm URL（例如 https://www.npmjs.com/package/@my-username/mcp-weather-server）来验证您的包已发布。

## 步骤 3：安装 `mcp-publisher`

Install the `mcp-publisher` CLI tool using a pre-built binary or [Homebrew](https://brew.sh):

<CodeGroup>

```bash macOS/Linux
curl -L "https://github.com/modelcontextprotocol/registry/releases/latest/download/mcp-publisher_$(uname -s | tr '[:upper:]' '[:lower:]')_$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/').tar.gz" | tar xz mcp-publisher && sudo mv mcp-publisher /usr/local/bin/
```

```powershell Windows
$arch = if ([System.Runtime.InteropServices.RuntimeInformation]::ProcessArchitecture -eq "Arm64") { "arm64" } else { "amd64" }; Invoke-WebRequest -Uri "https://github.com/modelcontextprotocol/registry/releases/latest/download/mcp-publisher_windows_$arch.tar.gz" -OutFile "mcp-publisher.tar.gz"; tar xf mcp-publisher.tar.gz mcp-publisher.exe; rm mcp-publisher.tar.gz
# Move mcp-publisher.exe to a directory in your PATH
```

```bash
brew install mcp-publisher
```

</CodeGroup>

通过运行以下命令验证 `mcp-publisher` 是否正确安装：

```bash
mcp-publisher --help
```

您应该看到如下输出：

```text Output
MCP Registry Publisher Tool

用法:
  mcp-publisher <command> [arguments]

命令:
  init          创建 server.json 文件模板
  login         向注册表进行身份验证
  logout        清除已保存的身份验证
  publish       将 server.json 发布到注册表
```

## 步骤 4：创建 `server.json`

`mcp-publisher init` 命令可以生成一个 `server.json` 模板文件，其中包含从您的项目派生的一些信息。

在您的服务器项目目录中，运行 `mcp-publisher init`：

```bash
mcp-publisher init
```

打开生成的 `server.json` 文件，您应该看到如下内容：

```json server.json
{
  "$schema": "https://static.modelcontextprotocol.io/schemas/2025-12-11/server.schema.json",
  "name": "io.github.my-username/weather",
  "description": "An MCP server for weather information.",
  "repository": {
    "url": "https://github.com/my-username/mcp-weather-server",
    "source": "github"
  },
  "version": "1.0.0",
  "packages": [
    {
      "registryType": "npm",
      "identifier": "@my-username/mcp-weather-server",
      "version": "1.0.0",
      "transport": {
        "type": "stdio"
      },
      "environmentVariables": [
        {
          "description": "Your API key for the service",
          "isRequired": true,
          "format": "string",
          "isSecret": true,
          "name": "YOUR_API_KEY"
        }
      ]
    }
  ]
}
```

根据需要编辑内容：

```diff server.json
 {
   "$schema": "https://static.modelcontextprotocol.io/schemas/2025-12-11/server.schema.json",
   "name": "io.github.my-username/weather",
   "description": "An MCP server for weather information.",
   "repository": {
     "url": "https://github.com/my-username/mcp-weather-server",
     "source": "github"
   },
-  "version": "1.0.0",
+  "version": "1.0.1",
   "packages": [
     {
       "registryType": "npm",
       "identifier": "@my-username/mcp-weather-server",
-      "version": "1.0.0",
+      "version": "1.0.1",
       "transport": {
         "type": "stdio"
-      },
-      "environmentVariables": [
-        {
-          "description": "Your API key for the service",
-          "isRequired": true,
-          "format": "string",
-          "isSecret": true,
-          "name": "YOUR_API_KEY"
-        }
-      ]
+      }
     }
   ]
 }
```

`server.json` 中的 `name` 属性**必须**与 `package.json` 中的 `mcpName` 属性匹配。

## 步骤 5：向 MCP 注册表进行身份验证

在本教程中，我们将使用基于 GitHub 的身份验证向 MCP 注册表进行身份验证。

运行 `mcp-publisher login` 命令以启动身份验证：

```bash
mcp-publisher login github
```

您应该看到如下输出：

```text Output
Logging in with github...

To authenticate, please:
1. Go to: https://github.com/login/device
2. Enter code: ABCD-1234
3. Authorize this application
Waiting for authorization...
```

访问链接，按照提示操作，并输入终端中打印的授权码（例如上述输出中的 `ABCD-1234`）。完成后，返回终端，您应该看到如下输出：

```text Output
Successfully authenticated!
✓ Successfully logged in
```

## 步骤 6：发布到 MCP 注册表

最后，使用 `mcp-publisher publish` 命令将您的服务器发布到 MCP 注册表：

```bash
mcp-publisher publish
```

您应该看到如下输出：

```text Output
Publishing to https://registry.modelcontextprotocol.io...
✓ Successfully published
✓ Server io.github.my-username/weather version 1.0.1
```

您可以通过使用 MCP 注册表 API 搜索来验证您的服务器是否已发布：

```bash
curl "https://registry.modelcontextprotocol.io/v0.1/servers?search=io.github.my-username/weather"
```

您应该在搜索结果 JSON 中看到您的服务器元数据：

```text Output
{"servers":[{ ... "name":"io.github.my-username/weather" ... }]}
```

## 故障排除

{/* prettier-ignore-start */}

| 错误信息 | 操作 |
| --- | --- |
|"Registry validation failed for package"|确保您的包包含所需的验证信息（例如 `package.json` 中的 `mcpName` 属性）。|
|"Invalid or expired Registry JWT token"|通过运行 `mcp-publisher login github` 重新进行身份验证。|
|"You do not have permission to publish this server"|您的身份验证方法与服务器的命名空间格式不匹配。使用 GitHub 身份验证时，您的服务器名称必须以 `io.github.your-username/` 开头。|

{/* prettier-ignore-end */}
