**太长不看版**：MCP 注册表相当宽松！我们只删除非法内容、恶意软件、垃圾信息和完全无法运行的服务器。

## 适用范围

此策略适用于位于 `registry.modelcontextprotocol.io` 的官方 MCP 注册表。

子注册表可能有自己的审核策略。如果您对特定子注册表上的内容有疑问，请直接联系他们。

## 免责声明

MCP 注册表**不**对审核做出保证，消费者应假设审核程度最低甚至没有。

MCP 注册表是一个社区支持的项目，我们的主动审核能力有限。我们在很大程度上依赖上游包注册表（如 NPM、PyPI 和 Docker）或下游子注册表（如 GitHub MCP 注册表）进行更深入的审核。

这意味着 MCP 注册表中可能存在根据本策略应被删除但尚未删除的内容。消费者应相应对待抓取的数据。

## 我们删除的内容

我们将删除包含以下内容的服务器：

- 非法内容，包括淫秽内容、版权侵犯和黑客工具
- 恶意软件，无论意图如何
- 垃圾信息，特别是批量创建的、扰乱注册表的服务器。例如：
  - 同一服务器以不同名称多次提交
  - 除了提供带有营销文案的固定响应外不做任何事情的服务器
  - 描述中塞满营销文案但实现不相关的服务器
- 无法正常运行的服务器

## 我们不删除的内容

总的来说，我们相信保持注册表开放并将审核推向子注册表。因此我们**不会**删除：

- 低质量或有 Bug 的服务器
- 存在安全漏洞的服务器
- 与其他服务器功能相同的服务器
- 提供或包含成人内容的服务器

## 删除流程

当我们删除服务器时，我们将服务器的 `status` 设置为 `"deleted"`，但服务器的元数据仍可通过 MCP 注册表 API 访问。聚合器随后可以从其索引中删除该服务器。

在极端情况下，我们可能会覆盖或擦除服务器的元数据。例如，如果元数据本身是非法的。

## 申诉

认为我们弄错了？在我们的 [GitHub 仓库](https://github.com/modelcontextprotocol/registry) 上创建一个 issue，包含：

- 服务器的名称
- 您认为该服务器不符合上述删除标准的理由

## 本策略的变更

我们仍在学习如何最好地运营 MCP 注册表！因此，我们将来可能会更改此策略。
