MCP 客户端和服务器之间的所有消息 **MUST** 遵循
[JSON-RPC 2.0](https://www.jsonrpc.org/specification) 规范。该协议定义了
三种基本消息类型：

| 类型            | 描述                   | 要求                     |
| --------------- | ---------------------- | ------------------------ |
| `Requests`      | 用于发起操作的消息     | 必须包含唯一 ID 和方法名 |
| `Responses`     | 作为请求回复发送的消息 | 必须包含与请求相同的 ID  |
| `Notifications` | 不需要回复的单向消息   | 不能包含 ID              |

**响应** 进一步分为 **成功结果** 或 **错误**。
结果可以遵循任何 JSON 对象结构，而错误至少必须包含错误码和消息。

## 协议层

Model Context Protocol 由几个协同工作的关键组件组成：

- **基础协议**：核心 JSON-RPC 消息类型
- **生命周期管理**：连接初始化、能力协商和会话控制
- **服务器特性**：服务器暴露的资源、提示和工具
- **客户端特性**：客户端提供的采样和根目录列表
- **实用工具**：日志记录和参数补全等横切关注点

所有实现 **MUST** 支持基础协议和生命周期管理组件。其他组件 **MAY** 根据应用程序的特定需求来实现。

这些协议层建立了清晰的关注点分离，同时实现了客户端和服务器之间的丰富交互。模块化设计允许实现精确支持它们所需的特性。

## 授权

身份验证和授权目前不是核心 MCP 规范的一部分，但我们正在考虑在将来引入它们的方式。加入我们的
[GitHub Discussions](https://github.com/modelcontextprotocol/specification/discussions)
一起塑造协议的未来！

客户端和服务器 **MAY** 协商它们自己的自定义身份验证和授权策略。

## 模式

协议的完整规范定义为
[TypeScript 模式](http://github.com/modelcontextprotocol/specification/tree/main/schema/2024-11-05/schema.ts)。
这是所有协议消息和结构的权威来源。

还有一个
[JSON Schema](http://github.com/modelcontextprotocol/specification/tree/main/schema/2024-11-05/schema.json)，
它是从 TypeScript 权威来源自动生成的，可用于各种自动化工具。
