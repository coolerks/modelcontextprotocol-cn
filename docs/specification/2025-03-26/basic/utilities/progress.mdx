---
title: 进度
---

Model Context Protocol (MCP) 支持通过通知消息可选地追踪长时间运行的操作的进度。任一方都可以发送进度通知，以提供操作状态更新。

## 进度流程

当一方想要*接收*请求的进度更新时，它在请求元数据中包含一个 `progressToken`。

- 进度令牌 **MUST** 是字符串或整数值
- 进度令牌可以由发送方以任何方式选择，但 **MUST** 在所有活动的请求中唯一。

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "some_method",
  "params": {
    "_meta": {
      "progressToken": "abc123"
    }
  }
}
```

接收方 **MAY** 然后发送包含以下内容的进度通知：

- 原始的进度令牌
- 当前的进度值
- 可选的"总量"值
- 可选的"消息"值

```json
{
  "jsonrpc": "2.0",
  "method": "notifications/progress",
  "params": {
    "progressToken": "abc123",
    "progress": 50,
    "total": 100,
    "message": "正在计算样条曲线..."
  }
}
```

- `progress` 值 **MUST** 随每次通知递增，即使总量未知。
- `progress` 和 `total` 值 **MAY** 是浮点数。
- `message` 字段 **SHOULD** 提供相关的人类可读进度信息。

## 行为要求

1. 进度通知 **MUST** 仅引用以下令牌：
   - 在活动请求中提供的
   - 与进行中的操作关联的

2. 进度请求的接收方 **MAY**：
   - 选择不发送任何进度通知
   - 以他们认为合适的任何频率发送通知
   - 如果总量未知则省略该值

```mermaid
sequenceDiagram
    participant Sender
    participant Receiver

    Note over Sender,Receiver: Request with progress token
    Sender->>Receiver: Method request with progressToken

    Note over Sender,Receiver: Progress updates
    loop Progress Updates
        Receiver-->>Sender: Progress notification (0.2/1.0)
        Receiver-->>Sender: Progress notification (0.6/1.0)
        Receiver-->>Sender: Progress notification (1.0/1.0)
    end

    Note over Sender,Receiver: Operation complete
    Receiver->>Sender: Method response
```

## 实现说明

- 发送方和接收方 **SHOULD** 追踪活动的进度令牌
- 双方 **SHOULD** 实现速率限制以防止洪泛
- 进度通知 **MUST** 在完成后停止发送
