本文档列出了自上一修订版 [2025-06-18](/specification/2025-06-18) 以来对 Model Context Protocol (MCP) 规范所做的变更。

## 主要变更

1. 增强授权服务器发现，支持 [OpenID Connect Discovery 1.0](https://openid.net/specs/openid-connect-discovery-1_0.html)。（PR [#797](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/797)）
2. 允许服务器将图标作为额外元数据暴露给工具、资源、资源模板和提示（[SEP-973](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/973)）。
3. 通过 `WWW-Authenticate` 增强授权流程，支持增量范围同意（[SEP-835](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/835)）
4. 提供关于工具命名的指南（[SEP-986](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/1603)）
5. 更新 `ElicitResult` 和 `EnumSchema`，使用更基于标准的方法，支持带标题、无标题、单选和多选枚举（[SEP-1330](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1330)）。
6. 增加对 [URL 模式征询](/specification/2025-11-25/client/elicitation#url-elicitation-requests) 的支持（[SEP-1036](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/887)）
7. 通过 `tools` 和 `toolChoice` 参数为采样添加工具调用支持（[SEP-1577](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1577)）
8. 增加对 OAuth 客户端 ID 元数据文档作为推荐客户端注册机制的支持（[SEP-991](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/991)，PR [#1296](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/1296)）
9. 增加对 [任务（tasks）](/specification/2025-11-25/basic/utilities/tasks) 的实验性支持，支持通过轮询和延迟结果检索跟踪持久请求（[SEP-1686](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1686)）。

## 次要变更

1. 澄清使用 stdio 传输的服务器可对 stderr 使用所有类型的日志记录，而不仅仅是错误消息（PR [#670](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/670)）。
2. 为 `Implementation` 接口添加可选的 `description` 字段，以对齐 MCP 注册表 server.json 格式，并在初始化期间提供人类可读的上下文。
3. 澄清在 Streamable HTTP 传输中，服务器必须对无效的 Origin 头部响应 HTTP 403 Forbidden。（PR [#1439](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/1439)）
4. 更新了 [安全最佳实践指南](https://modelcontextprotocol.io/specification/draft/basic/security_best_practices)。
5. 澄清输入验证错误应作为工具执行错误返回，而非协议错误，以便模型能够自我纠正（[SEP-1303](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1303)）。
6. 通过允许服务器随意断开连接来支持轮询 SSE 流（[SEP-1699](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1699)）。
7. 澄清 SEP-1699：GET 流支持轮询，无论流来源如何始终通过 GET 恢复，事件 ID 应对流身份进行编码，断开连接包括服务器发起的关闭（问题 [#1847](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1847)）。
8. 使 OAuth 2.0 受保护资源元数据发现与 RFC 9728 对齐，将 `WWW-Authenticate` 头部设为可选，并回退到 `.well-known` 端点（[SEP-985](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/985)）。
9. 为征询模式的所有基本类型（字符串、数字、枚举）添加默认值支持（[SEP-1034](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1034)）。
10. 将 JSON Schema 2020-12 确立为 MCP schema 定义的默认方言（[SEP-1613](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1613)）。

## 其他 schema 变更

1. 将请求负载与 RPC 方法定义解耦为独立的参数 schemas。（[SEP-1319](https://github.com/modelcontextprotocol/specification/issues/1319)，PR [#1284](https://github.com/modelcontextprotocol/specification/pull/1284)）

## 治理和流程更新

1. 正式确立 Model Context Protocol 治理结构（[SEP-932](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/932)）。
2. 为 MCP 社区建立共享的沟通实践和指南（[SEP-994](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/994)）。
3. 在 MCP 治理中正式确立工作组和兴趣组（[SEP-1302](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1302)）。
4. 建立 SDK 分层体系，明确功能支持和维护承诺的要求（[SEP-1730](https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1730)）。
